# Monorepo Structure

Eco uses a structured monorepo approach with clear separation of concerns:

## Apps Layer
- **Shell**: Main application shell and navigation
- **Docs**: Documentation Power Page
- **Tracker**: Resource tracking interface
- **Marketplace**: Plugin and extension marketplace
- **Assist**: Front-end companion agent

## Packages Layer
- **Core Tokens**: Design system tokens and CSS variables
- **Halo UI**: Cinematic UI components
- **Shared Types**: TypeScript type definitions
- **Core**: Shared utilities and business logic

## Agents Layer
- Specialized workers for various tasks
- Orchestrated by the workflow engine
- Maintain provenance throughout operations
